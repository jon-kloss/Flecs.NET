# Flecs.NET Upstream Parity & .NET 10 Upgrade Plan

## Goal
Bring Flecs.NET up to parity with upstream flecs v4.1.5 (currently at v4.0.4) and upgrade the target framework from .NET 9 to .NET 10.

## Current State
- Flecs.NET version: 4.0.4
- Upstream flecs version: 4.1.5 (released 2026-03-15)
- Releases to catch up: v4.0.5, v4.1.0, v4.1.1, v4.1.2, v4.1.3, v4.1.4, v4.1.5
- Target framework: net9.0

--------------------------------------------------------------------------------

## Phase 1: .NET 10 Upgrade

.NET 10 is fully released (LTS, latest 10.0.6, end of support Nov 2028).

### 1.1 Update Target Frameworks
- Update all .csproj files from net9.0 to net10.0 (7 projects)
- Keep Flecs.NET.Native at netstandard2.1 (native-only package, no managed code)
- Update SDK version in global.json if present

### 1.2 Verify Compatibility
- Run full test suite — unsafe code and P/Invoke behavior unchanged in .NET 10
- Check if any analyzer warnings change (EnforceCodeStyleInBuild is enabled)
- Verify Zig cross-compilation still works with .NET 10 SDK

--------------------------------------------------------------------------------

## Phase 2: Update Native Submodule to v4.1.5

### 2.1 Update Submodule
- Update native/flecs submodule to v4.1.5 tag
- Verify Zig cross-compilation still works for all targets

  CONTEXT: UPSTREAM CHANGELOG HIGHLIGHTS
================================================================================

v4.0.5 — Performance/memory focused
  - ecs_shrink/world::shrink for freeing unused memory
  - Inheritable trait
  - Equals/compare type hooks in meta
  - Script parser split from FLECS_SCRIPT addon

v4.1.0 — MAJOR release
  - Union relationships REMOVED (replaced by DontFragment + Exclusive)
  - Non-fragmenting components (DontFragment trait)
  - World exclusive access
  - OrderedChildren storage
  - Table C++ API expanded significantly
  - get() returns const T&, get_mut() returns T& (new try_get/try_get_mut)
  - entity::child(), entity::assign()
  - Single-term observers without query
  - Bloom filter for queries
  - Removed entity id creation in multithreaded mode

v4.1.1 — Performance + new hook
  - on_replace hook (new lifecycle hook!)
  - Singleton trait
  - New size argument to ecs_emplace_id/ecs_ensure_id
  - world::id_if_registered, world::type_info()
  - EcsConstants component split from EcsEnum/EcsBitmask

v4.1.2 — C++17 requirement + memory stats
  - C++ now requires C++17 (not relevant for C# wrapper)
  - Detailed memory statistics API
  - New REST endpoints: /components, /queries
  - Any wildcard cacheable
  - Observer desc observable field removed

v4.1.3 — RAM utilization
  - Fewer component records created
  - ecs_is_defer_suspended() function
  - ecs_record_t::cr field removed

v4.1.4 — Bugfix release
  - ecs_get_version() function

v4.1.5 — Hierarchy storage + script math
  - Non-fragmenting hierarchy storage (Parent relationship)
  - world.each(callback) for iterating all entities
  - System component deduction from each() arguments
  - Math/Perlin noise in script
  - Vector type operations in script

Well-covered areas:
  - World, Entity, Component, Query, System, Observer, Pipeline (core ECS)
  - QueryBuilder, ObserverBuilder, SystemBuilder, PipelineBuilder (builders)
  - Iter, Field, FieldData, Ref (iteration)
  - Alert, AlertBuilder, MetricBuilder (addons)
  - Doc module (on Entity + Ecs.Doc class)
  - JSON serialization (ToJson/FromJson on Entity)
  - ScriptBuilder (basic script loading)
  - AppBuilder (with REST enabling)
  - Cursor (meta cursor API)
  - Type hook interfaces (Ctor, Dtor, Copy, Move, OnAdd, OnRemove, OnSet)
  - Code generation for 1-16 generic type parameters
  - 946 native function bindings in Flecs.g.cs
  - 80+ working examples

Modules registered: Alerts, Meta (primitive types only), Metrics, Stats, Units

Architecture:
  - Bindings auto-generated from C headers via Bindgen.NET
  - Native lib cross-compiled via Zig toolsets
  - Code generator produces generic overloads (T0..T15)
  - All core types are structs wrapping native pointers
  - BindingContext system manages GC-preventing of delegates passed to native code
